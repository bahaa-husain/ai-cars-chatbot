GOAL
----
Align the Python chatbot layer with the C# car backend entity and enums,
so both sides share the same field names, enum names, and integer values.
No files inside "cars backend/" were modified.

----------------------------------------
1. NEW FILE: enums.py
----------------------------------------
Created a new module containing six IntEnum classes that mirror the C#
enums in cars backend/App_backend/Models/Enums/CarEnums/ exactly.

  BodyType        (0=Sedan, 1=Hatchback, 2=Coupe, 3=SUV, 4=Bus, 5=MiniTruck)
  CarCondition    (0=Used, 1=New)
  CarReactionType (0=Like, 1=Dislike)
  CarStatus       (0=Draft, 1=PendingReview, 2=Available, 3=NotAvailable,
                   4=Rejected, 5=Sold, 6=Archived)
  FuelType        (0=Gas, 1=Diesel, 2=Electric, 3=Hybrid)
  GearType        (0=Auto, 1=Manual)

2. CHANGED FILE: mock_data.py
----------------------------------------

Imports:
  + Added: from enums import BodyType, CarCondition, CarStatus, FuelType, GearType

Car dictionary fields - RENAMED:
  name              -> title
  price             -> price_usd  (USD value, approx SYP/15000)
                    -> price_syp  (original SYP value kept under new name)
  views             -> clicks
  favorites         -> favorite_count
  market_avg_price  -> average_model_price
  image             -> main_photo_url

Car dictionary fields - ADDED (were missing from old mock):
  business_account_id  (int, unique per listing)
  model                (car model string, e.g. "Camry", "Sportage")
  status               (CarStatus.Available for all mock entries)
  condition            (CarCondition.Used for all mock entries)
  gear_type            (GearType.Auto for all mock entries)
  doors                (int, 4 for all mock entries)
  interior_color       (Arabic color string)
  likes                (int, separate from favorite_count)
  dislikes             (int)

Car dictionary fields - REMOVED (not present in C# Car entity):
  location             (was Arabic city string)
  seats                (was integer seat count)

Enum field values updated:
  fuel_type values changed from Arabic strings ("benzene") to FuelType enum members
  body_type values changed from lowercase strings ("sedan","suv") to BodyType enum members

id field type:
  Changed from string ("1","2",...) to int (1, 2, ...) to match C# int Id

year / mileage field types:
  Changed from int to string to match C# string Year / string Mileage

Function: search_mock_cars()
  - Removed parameter: location (no longer in Car entity)
  - budget filter now compares against price_syp instead of price
  - body_type filter now uses BodyType[name] enum lookup instead of string contains
  - sort key updated: year/mileage cast to int (now stored as strings),
    favorites -> favorite_count

Function: get_market_analysis()
